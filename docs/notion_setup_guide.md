# Notion 準備ガイド（管理者ダッシュボード連携）

このアプリの「管理者側」をNotionで作るための手順です。所要 約10分。
専門知識は不要。上から順にやればOKです。

> ねらい：現場スタッフの入力が、Notion上の1枚の表に自動でまとまり、
> 経営層・管理者はそこを見るだけで全会場の進捗と報告用データを把握できる状態にする。

---

## 全体像

```
現場アプリ → Supabase(本体DB) → 【Notionの表】← 管理者はここを見るだけ
```

Notion側に用意するのは「表（データベース）1つ」と「連携キー（トークン）」です。

---

## ステップ1：インテグレーション（連携アプリ）を作る

1. ブラウザで https://www.notion.so/my-integrations を開く
2. **「New integration」** をクリック
3. 入力：
   - Name：`event-sales-sync`（任意の名前でOK）
   - Associated workspace：使うワークスペースを選択
   - Type：Internal
4. **「Submit」** → 作成後の画面で **「Internal Integration Secret」** をコピー
   - `ntn_...` または `secret_...` で始まる文字列
   - 👉 これが **token**。あとでDaigoroに共有（＝secretsに設定）

> 🔑 トークンは「Notionを外部から操作するための鍵」です。人に見せない・SNSに貼らない。

---

## ステップ2：集約用データベース（表）を作る

1. Notionで新しいページを作成 → `/database` と打って **「Table - Full page」** を選ぶ
2. タイトルを付ける（例：`イベント実績（自動同期）`）
3. 下記の **プロパティ（列）を正確に同じ名前・同じ型** で作る
   （名前が1文字でも違うと同期がずれるので注意）

| プロパティ名 | 型（Type） | 備考 |
|---|---|---|
| `会場` | Title（タイトル） | 最初からある「Name」列をリネームでOK |
| `担当者` | Text（テキスト） | スタッフ人数 |
| `イベント期間` | Date（日付） | 開始〜終了 |
| `期間目標値` | Number（数値） | 新規＋MNPの期間目標 |
| `MNP` | Number（数値） | 固定KPI |
| `新規契約` | Number（数値） | 固定KPI |
| `内訳` | Text（テキスト） | 自由項目を「機種変更×5, クレカ×3」形式で集約 |
| `合計` | Number（数値） | 全項目合計 |
| `更新時刻` | Text（テキスト） | |
| `_key` | Text（テキスト） | 会場×期間の一意キー（アプリが内部利用・非表示推奨） |

> 💡 `_key` は「会場｜開始日｜終了日」を覚えておく目印です。
> これがあるおかげで、同じイベントの行が二重に作られず、毎回きれいに上書き更新されます。
> 管理者は普段は見なくてOK（非表示にしても動きます）。

---

## ステップ3：作った表にインテグレーションを接続する

1. 作成した表のページ右上の **「•••」（3点メニュー）** をクリック
2. **「Connections」→「Connect to」**（または「コネクトの追加」）を選ぶ
3. ステップ1で作った **`event-sales-sync`** を選んで接続

> ⚠️ この接続を忘れると「権限がありません」エラーになります。ここが一番の落とし穴。

---

## ステップ4：データベースIDを控える

1. 作った表を **ページ全体表示** で開く
2. ブラウザのURLを見る。形はこんな感じ：
   ```
   https://www.notion.so/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx?v=yyyy...
                          └──────────── ここ32文字 ────────────┘
   ```
3. `notion.so/` の直後〜`?` の前までの **32文字の英数字** が **database_id**
   - ワークスペース名が入る場合は、その後ろの32文字部分
   - 👉 これをDaigoroに共有

---

## Daigoroに渡すもの（まとめ）

| 項目 | 例 | どこで取得 |
|---|---|---|
| token | `ntn_xxxxxxxx...` | ステップ1 |
| database_id | `1a2b3c4d...`（32文字） | ステップ4 |

これらは `.streamlit/secrets.toml` の `[notion]` に設定します
（コードには書かない・GitHubにも上げない）。設定はDaigoroが行います。

```toml
[notion]
token       = "ntn_xxxxxxxx..."
database_id = "1a2b3c4d..."
```

---

## 動作確認（S2疎通テスト）

キー設定後、アプリの下部に出る
**「🗂 管理者用：Notionへ同期」→「今すぐNotionに同期する」** を押すと、
現在の会場のスタッフ別合計がNotionの表に反映されます。
（本番では、この同期を数分おきに自動実行する仕組み＝定期同期をS2の仕上げで追加します）

---

## よくあるつまづき

- **「Could not find database」** → ステップ3の接続忘れ、またはdatabase_idの桁ずれ
- **数字が入らない/文字化け** → プロパティ名 or 型の不一致（表の列名を再確認）
- **行が二重にできる** → `_key` プロパティが未作成、または型がTextでない
